## 迁移概述

### 项目组件

| 组件 | 说明 | 迁移内容 |
|-----|------|---------|
| **后端服务** | Spring Boot 3.2 + Java 21 | 源代码 + 配置文件 |
| **小程序前端** | 微信原生小程序 | 源代码 |
| **管理后台** | React + Vite | 源代码 + node_modules |
| **数据库** | MySQL 8.0 | 数据库结构 + 数据 |
| **上传文件** | 用户上传的图片等 | upload 目录 |

### 迁移方式选择

| 方式 | 适用场景 | 优点 | 缺点 |
|-----|---------|------|------|
| **压缩包传输** | 小团队/个人 | 简单直接 | 文件较大 |
| **Git 克隆** | 有版本控制 | 可获取最新代码 | 需额外导出数据 |
| **Docker 部署** | 生产环境 | 环境一致性好 | 需要 Docker 知识 |

## 完整迁移步骤

### 步骤 1: 目标电脑环境准备

在新电脑上安装必需软件：

| 软件 | 版本 | 下载地址 | 验证命令 |
|-----|------|---------|---------|
| JDK | 21+ | https://adoptium.net/ | `java -version` |
| Maven | 3.8+ | https://maven.apache.org/ | `mvn -version` |
| MySQL | 8.0+ | https://dev.mysql.com/downloads/ | `mysql --version` |
| Node.js | 18+ | https://nodejs.org/ | `node -v` |
| 微信开发者工具 | 最新 | https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html | - |

#### 配置环境变量

**Windows 环境变量配置**：

```powershell
# 以管理员身份运行 PowerShell

# 设置 JAVA_HOME
[System.Environment]::SetEnvironmentVariable("JAVA_HOME", "C:\Program Files\Eclipse Adoptium\jdk-21", "Machine")

# 设置 MAVEN_HOME
[System.Environment]::SetEnvironmentVariable("MAVEN_HOME", "C:\apache-maven-3.9.6", "Machine")

# 添加到 PATH
$oldPath = [System.Environment]::GetEnvironmentVariable("Path", "Machine")
$newPath = $oldPath + ";%JAVA_HOME%\bin;%MAVEN_HOME%\bin"
[System.Environment]::SetEnvironmentVariable("Path", $newPath, "Machine")
```

> ⚠️ 配置完成后，重新打开终端窗口以使环境变量生效。

### 步骤 2: 解压项目文件

```powershell
# 解压到目标目录（例如 D:\Projects）
cd D:\Projects
7z x 乡村农产品直销平台.7z

# 或使用 Windows 资源管理器右键解压
```

### 步骤 3: 配置数据库

#### 3.1 启动 MySQL 服务

```powershell
# Windows 服务启动
net start mysql

# 或通过 MySQL Installer 启动
```

#### 3.2 创建数据库

```sql
-- 登录 MySQL
mysql -u root -p

-- 创建数据库
CREATE DATABASE village CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 查看数据库列表
SHOW DATABASES;
```

#### 3.3 导入数据

```powershell
# 使用导出的备份文件
mysql -u root -p village < D:\Projects\乡村农产品直销平台---乡村农产品直销平台\village_backup.sql

# 或使用项目自带的初始化脚本（如果没有备份文件）
mysql -u root -p village < D:\Projects\乡村农产品直销平台---乡村农产品直销平台\village\src\main\resources\village.sql
```

#### 3.4 验证数据导入

```sql
USE village;
SHOW TABLES;
SELECT COUNT(*) FROM user;
SELECT COUNT(*) FROM product;
```

### 步骤 4: 配置后端

#### 4.1 修改数据库连接配置

编辑 `village/src/main/resources/application.yml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/village?useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true&characterEncoding=UTF-8
    username: root
    password: YOUR_MYSQL_PASSWORD  # 改为新电脑的 MySQL 密码
    driver-class-name: com.mysql.cj.jdbc.Driver
```

#### 4.2 配置上传路径（可选）

如果需要修改上传文件存储路径：

```yaml
upload:
  path: D:/Projects/乡村农产品直销平台---乡村农产品直销平台/village/upload/
```

### 步骤 5: 启动后端服务

```powershell
# 进入后端目录
cd D:\Projects\乡村农产品直销平台---乡村农产品直销平台\village

# 首次运行需要下载依赖（可能需要几分钟）
mvn clean package -DskipTests

# 启动服务
mvn spring-boot:run
```

**预期输出**：
```

║   🌾 乡村农产品直销平台后端服务启动成功！                                ║
╚═══════════════════════════════════════════════════════════╝
```

### 步骤 6: 配置并启动管理后台

```powershell

cd D:\Projects\乡村农产品直销平台---乡村农产品直销平台\village\admin-frontend

# 安装依赖

npm run dev
```

**预期输出**：
```

```

### 步骤 7: 配置并启动小程序

#### 7.1 打开微信开发者工具

1. 启动微信开发者工具
2. 选择 **导入项目**
3. 目录选择：`D:\Projects\乡村农产品直销平台---乡村农产品直销平台\miniprogram`
4. AppID：使用测试号或正式 AppID

#### 7.2 配置 API 地址

如果后端运行在不同地址，修改 `miniprogram/utils/request.js`：

```javascript
// 默认配置 - 本地开发
const baseURL = 'http://127.0.0.1:8080/api';

// 如果后端在其他 IP
const baseURL = 'http://192.168.1.100:8080/api';
```

#### 7.3 关闭域名校验

**详情 → 本地设置 → 勾选 "不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书"**

## 配置调整

### 根据新环境调整的主要配置项

| 文件 | 配置项 | 说明 |
|-----|--------|------|
| `application.yml` | `spring.datasource.password` | MySQL 密码 |
| `application.yml` | `upload.path` | 文件上传路径 |
| `request.js` | `baseURL` | 后端 API 地址 |
| `vite.config.js` | `server.proxy.target` | API 代理地址（如有） |

### IP 地址变更配置

如果后端不再是 `127.0.0.1`：

1. **小程序 `request.js`**：
   ```javascript
   const baseURL = 'http://新IP地址:8080/api';
   ```

2. **小程序图片 URL 格式化**：
   ```javascript
   // 在 formatImageURL 函数中
   if (url.indexOf('/uploads/') === 0) {
       const fullURL = 'http://新IP地址:8080' + url;
       return fullURL;
   }
   ```

3. **管理后台 API 配置**（如果单独配置了）：
   检查 `admin-frontend/src/api/` 或 `admin-frontend/src/utils/` 下的请求配置

## 不同场景迁移指南

### 场景 A: 同一局域网内迁移

最简单的场景，直接复制文件即可。

```powershell
# 通过网络共享复制
xcopy \\源电脑\共享文件夹\乡村农产品直销平台.7z D:\Projects\ /Y
```

### 场景 B: 迁移到公网服务器

需要额外配置：

1. **开放端口**：
   - 8080（后端 API）
   - 3306（MySQL，如外部访问）

2. **配置 CORS**：
   `SecurityConfig.java` 中已配置允许所有来源

3. **修改小程序请求地址**：
   ```javascript
   const baseURL = 'http://服务器公网IP:8080/api';
   ```

4. **上传到服务器**：
   ```bash
   scp 乡村农产品直销平台.7z user@server:/home/user/
   ```

### 场景 C: Docker 容器化部署

创建 `docker-compose.yml`：

```yaml
version: '3.8'
services:
  mysql:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: 123456
      MYSQL_DATABASE: village
    volumes:
      - mysql_data:/var/lib/mysql
      - ./village.sql:/docker-entrypoint-initdb.d/init.sql
    ports:
      - "3306:3306"

  backend:
    build: ./village
    ports:
      - "8080:8080"
    depends_on:
      - mysql
    environment:
      SPRING_DATASOURCE_URL: jdbc:mysql://mysql:3306/village
      SPRING_DATASOURCE_PASSWORD: 123456

volumes:
  mysql_data:
```

后端 `Dockerfile`：
```dockerfile
FROM eclipse-temurin:21-jdk
WORKDIR /app
COPY target/village-api-1.0.0.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar"]
```

## 常见问题排查

### Q1: Maven 依赖下载失败

**症状**：`mvn clean package` 卡住或报错

**解决**：配置阿里云镜像

在用户目录下创建/编辑 `~/.m2/settings.xml`：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<settings>
  <mirrors>
    <mirror>
      <id>aliyunmaven</id>
      <mirrorOf>*</mirrorOf>
      <name>阿里云公共仓库</name>
      <url>https://maven.aliyun.com/repository/public</url>
    </mirror>
  </mirrors>
</settings>
```

### Q2: npm install 失败

**解决**：

```powershell
# 清理缓存
npm cache clean --force

# 删除 node_modules
rmdir /s /q node_modules

# 使用淘宝镜像
npm install --registry=https://registry.npmmirror.com
```

### Q3: 数据库连接失败

**检查项**：
1. MySQL 服务是否启动：`net start mysql`
2. 数据库是否存在：`SHOW DATABASES;`
3. 密码是否正确
4. 确认 `application.yml` 配置

### Q4: 图片无法显示

**检查项**：
1. `upload` 目录是否复制完整
2. 文件路径是否正确
3. 后端静态资源配置是否正确

**测试**：
```
直接访问: http://localhost:8080/uploads/consumer.jpg
```

### Q5: 小程序报网络错误

**检查项**：
1. 后端服务是否运行
2. `request.js` 中的 `baseURL` 是否正确
3. 域名校验是否关闭
4. 网络防火墙是否阻止

### Q6: Java 版本不匹配

**症状**：`UnsupportedClassVersionError`

**解决**：安装 JDK 21 并配置 `JAVA_HOME`

```powershell
java -version
# 确认输出为 21.x.x
```

### Q7: 端口冲突

**解决**：

```powershell
# 查看端口占用
netstat -ano | findstr :8080
netstat -ano | findstr :5173

# 终止占用进程
taskkill /PID <进程ID> /F
```

## 迁移脚本

### 自动化迁移脚本 (Windows)

创建 `migrate.bat`：

```batch
@echo off
chcp 65001
echo ========================================
echo   乡村农产品直销平台项目迁移助手
echo ========================================
echo.

:: 检查环境
echo [1/6] 检查环境...
java -version 2>nul
if errorlevel 1 (
    echo 错误: 未安装 Java，请先安装 JDK 21
    pause
    exit /b 1
)

mvn -version 2>nul
if errorlevel 1 (
    echo 错误: 未安装 Maven
    pause
    exit /b 1
)

node -v 2>nul
if errorlevel 1 (
    echo 错误: 未安装 Node.js
    pause
    exit /b 1
)

echo 环境检查通过！
echo.

:: 设置项目目录（根据实际情况修改）
set PROJECT_DIR=%~dp0
echo 项目目录: %PROJECT_DIR%
echo.

:: 构建后端
echo [2/6] 构建后端项目...
cd /d "%PROJECT_DIR%village"
call mvn clean package -DskipTests
if errorlevel 1 (
    echo 后端构建失败！
    pause
    exit /b 1
)
echo 后端构建成功！
echo.

:: 安装前端依赖
echo [3/6] 安装管理后台依赖...
cd /d "%PROJECT_DIR%village\admin-frontend"
call npm install --registry=https://registry.npmmirror.com
echo.

:: 提示数据库配置
echo [4/6] 数据库配置
echo ----------------------------------------
echo 请确保已完成以下操作：
echo 1. MySQL 服务已启动
echo 2. 已创建 village 数据库
echo 3. 已导入数据（使用 village.sql）
echo 4. application.yml 中的密码已正确配置
echo ----------------------------------------
echo.
pause

:: 启动服务
echo [5/6] 是否启动服务？(Y/N)
set /p START_SERVICES=

if /i "%START_SERVICES%"=="Y" (
    echo 启动后端服务...
    start cmd /k "cd /d %PROJECT_DIR%village && mvn spring-boot:run"
    
    timeout /t 15 /nobreak
    
    echo 启动管理后台...
    start cmd /k "cd /d %PROJECT_DIR%village\admin-frontend && npm run dev"
)

echo.
echo [6/6] 迁移完成！
echo ========================================
echo 服务地址：
echo   后端 API: http://localhost:8080
echo   管理后台: http://localhost:5173
echo   API 文档: http://localhost:8080/swagger-ui/index.html
echo ========================================
echo.
echo 请手动打开微信开发者工具导入小程序项目。
echo 小程序目录: %PROJECT_DIR%miniprogram
echo.
pause
```
