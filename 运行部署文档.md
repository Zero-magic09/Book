## 环境要求

在开始之前，请确保您的开发环境已安装以下软件：

## 数据库配置

### 步骤 1: 创建数据库

登录 MySQL 并创建数据库：

```sql
-- 登录 MySQL
mysql -u root -p

-- 创建数据库
CREATE DATABASE village CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 验证创建
SHOW DATABASES;
```

### 步骤 2: 导入初始数据

使用提供的 SQL 脚本初始化数据库：

```powershell
# 方式一：命令行导入
mysql -u root -p village < d:\IDEA\乡村农产品直销平台---乡村农产品直销平台\village\src\main\resources\village.sql

# 方式二：使用 MySQL 客户端工具（如 Navicat, DBeaver）
# 1. 连接到 MySQL
# 2. 选择 village 数据库
# 3. 运行 SQL 文件: village\src\main\resources\village.sql
```

### 步骤 3: 验证数据导入

```sql
USE village;
SHOW TABLES;
```

预期输出应包含以下表：
```
address
bank_account
cart
coupon
farm_photo
farmer
order_item
orders
product
review
user
user_coupon
user_favorite
withdrawal
```

### 数据库连接配置

后端默认配置（位于 `village/src/main/resources/application.yml`）：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/village?useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true&characterEncoding=UTF-8
    username: root
    password: 123456
    driver-class-name: com.mysql.cj.jdbc.Driver
```

> ⚠️ **注意**：如果您的 MySQL 密码不是 `123456`，请修改 `application.yml` 中的 `password` 字段。

## 后端服务启动

### 方式一：使用 Maven 命令行

```powershell
# 进入项目目录
cd d:\IDEA\乡村农产品直销平台---乡村农产品直销平台\village

# 清理并编译项目
mvn clean package -DskipTests

# 运行项目
mvn spring-boot:run

# 或者直接运行 JAR 包
java -jar target/village-api-1.0.0.jar
```

### 方式二：使用 IDE (IntelliJ IDEA)

1. 使用 IntelliJ IDEA 打开 `village` 目录
2. 等待 Maven 依赖下载完成
3. 找到 `VillageApplication.java` 文件
4. 右键点击 → `Run 'VillageApplication'`

## 微信小程序启动

### 步骤 1: 打开微信开发者工具

1. 启动微信开发者工具
2. 选择 **导入项目**
3. 项目目录选择：`d:\IDEA\乡村农产品直销平台---乡村农产品直销平台\miniprogram`
4. AppID 可以选择"测试号"（如果没有正式 AppID）

### 步骤 2: 配置项目

项目配置文件 `project.config.json` 已预设好：

```json
{
    "appid": "wxXXXXXXXXX",
    "projectname": "乡村农产品直销平台",
    "setting": {
        "urlCheck": false,
        "es6": true,
        "enhance": true
    }
}
```

> ⚠️ **重要**：由于小程序需要连接本地后端 API (`http://127.0.0.1:8080`)，需要在开发者工具中关闭域名校验：
> 
> **详情 → 本地设置 → 勾选 "不校验合法域名..."**

### 步骤 3: 编译运行

1. 点击工具栏的 **编译** 按钮
2. 模拟器中将显示小程序首页（角色选择页面）

### API 地址配置

如果后端运行在其他地址，修改 `miniprogram/utils/request.js`：

```javascript
// 修改为实际的后端地址
const baseURL = 'http://127.0.0.1:8080/api';
```

## 常见问题

### Q1: 后端启动失败，提示数据库连接错误

**可能原因**：
- MySQL 服务未启动
- 数据库名称、用户名或密码错误

**解决方案**：
```powershell
# 检查 MySQL 服务状态
net start mysql

# 确认可以登录
mysql -u root -p

# 检查数据库是否存在
SHOW DATABASES;
```

修改 `application.yml` 中的数据库配置。

### Q2: 端口 8080 被占用

**解决方案**：
```powershell
# 查找占用端口的进程
netstat -ano | findstr :8080

# 终止进程（替换 PID）
taskkill /PID <PID> /F

# 或修改 application.yml 中的端口
server:
  port: 8081
```

### Q3: 小程序无法连接后端

**可能原因**：
- 后端未启动
- 域名校验未关闭
- API 地址配置错误

**解决方案**：
1. 确保后端服务正在运行
2. 在微信开发者工具中：**详情 → 本地设置 → 勾选 "不校验合法域名..."**
3. 检查 `request.js` 中的 `baseURL`

### Q4: 管理后台 `npm install` 失败

**解决方案**：
```powershell
# 清理缓存
npm cache clean --force

# 删除 node_modules 和 package-lock.json
rmdir /s /q node_modules
del package-lock.json

# 使用淘宝镜像重新安装
npm install --registry=https://registry.npmmirror.com
```

### Q5: 图片无法显示

**可能原因**：
- 上传目录不存在或无权限
- 静态资源路径配置错误

**解决方案**：
1. 确保 `village/upload` 目录存在
2. 检查 `application.yml` 中的 `upload.path` 配置
3. 确认图片文件确实存在

### Q6: Maven 依赖下载缓慢

**解决方案**：配置 Maven 使用阿里云镜像

在 `~/.m2/settings.xml` 或 `MAVEN_HOME/conf/settings.xml` 中添加：

```xml
<mirrors>
    <mirror>
        <id>aliyunmaven</id>
        <mirrorOf>*</mirrorOf>
        <name>阿里云公共仓库</name>
        <url>https://maven.aliyun.com/repository/public</url>
    </mirror>
</mirrors>
```
